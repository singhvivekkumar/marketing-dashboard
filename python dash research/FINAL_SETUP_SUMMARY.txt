Edit .env file to customize:

  DASH_DEBUG=True              # Debug mode enabled
  DASH_HOST=0.0.0.0          # Network accessible
  DASH_PORT=8050             # Port number
  DATA_FOLDER=src/Leads_Data # Data location

COMMON CHANGES:

  Change port (if 8050 busy):
    DASH_PORT=8051

  Disable debug mode (production):
    DASH_DEBUG=False

  Localhost only:
    DASH_HOST=127.0.0.1

┌──────────────────────────────────────────────────────────────────────────────┐
│ 🆘 TROUBLESHOOTING QUICK FIX                                                 │
└──────────────────────────────────────────────────────────────────────────────┘

PROBLEM: "No module named 'dash'"
SOLUTION: pip install -r requirements.txt

PROBLEM: "Port 8050 already in use"
SOLUTION: Change DASH_PORT in .env file

PROBLEM: "No data showing"
SOLUTION: 
  1. Check Excel files in src/Leads_Data/
  2. Verify column names (case-sensitive)
  3. Restart: python app.py

PROBLEM: "Permission denied when activating venv"
SOLUTION: Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

PROBLEM: "venv not found"
SOLUTION: python -m venv venv
