Edit .env to customize:

  # Debug mode (shows errors, live reload)
  DASH_DEBUG=True

  # Server host (0.0.0.0 = accessible from network)
  DASH_HOST=0.0.0.0

  # Port number
  DASH_PORT=8050

  # Data folder location
  DATA_FOLDER=src/Leads_Data

COMMON CUSTOMIZATIONS:

  Change port (if 8050 busy):
    DASH_PORT=8051

  Disable debug mode (for production):
    DASH_DEBUG=False

  Restrict to localhost only:
    DASH_HOST=127.0.0.1

PROBLEM: "No module named 'dash'"
вҶ“
SOLUTION:
  venv\Scripts\Activate.ps1
  pip install -r requirements.txt

PROBLEM: "Port 8050 already in use"
вҶ“
SOLUTION:
  Edit .env and change:
  DASH_PORT=8051

PROBLEM: "No data showing in dashboard"
вҶ“
SOLUTION:
  1. Check Excel files are in src/Leads_Data/
  2. Verify column names match exactly (case-sensitive)
  3. Check for empty sheets
  4. Restart: python app.py
  5. Check browser console: F12

PROBLEM: "Permission denied when activating venv"
вҶ“
SOLUTION:
  Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
  venv\Scripts\Activate.ps1

PROBLEM: "python command not found"
вҶ“
SOLUTION:
  1. Install Python from python.org
  2. Add to PATH: https://docs.python.org/3/using/windows.html
  3. Restart terminal

PROBLEM: "Dates showing incorrectly"
вҶ“
SOLUTION:
  Ensure Excel dates are in format: YYYY-MM-DD
  Or let Excel format them naturally
